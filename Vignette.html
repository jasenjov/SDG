This case of use will be based on generating and comparing the predictive performance of students' results data generated according to different models. This library allows the generation of data using 7 different models that will be described in more detail later. So the point of this case of use will be to generate data using the provided functions and according to different models. After this, we will generate performance data using a specific underlying model and we will analyse the predictive performance of the other models. This will be clarified farther with particular examples. All the data used will be synthetic data, generated according to our needs and using the library's functions.

So, the first step is to load the library we are working with: Synthetic Data Generation.

library(SDG)

Once we have imported all the functionalities of the SDG package, the example of use starts by generating a matrix involving 10 students and 4 skills. Each element will represent the probability of a student to succeed an item that calls for the correspondent skill. This information is generated according to the Gaussian function.

The function used is skillsGen(), that generates data according to the standard distribution, as mentioned before. We will consider a mean of 0 and a standard deviation of 1. If we would like to introduce a noise while predicting the students' skills, we could modify the standard deviation of the Gaussian function by changing the 'deviation' parameter. The matrix generated here will be the skills matrix used along the whole document.

skills <- skillsGen(sts = 10,mean = 0, deviation = 1,rank = 4)

Once we have the skills matrix, we need the information about the set of items that the students are going to answer. So we generate this group of items using a Q-Matrix involving 4 skills, according to the number of skills defined while generating the skills matrix. In order to get this kind of data, the QgenInc() function is used, in which the 'maxSkillsPerItem' parameter will define the number of items generated. We set it to 4, which means that we will have a number of items corresponding to all the combinations, involving maximum 4 skills. We start generating data according to the pessimistic approach. To better understand this model, we are going to suppose an item involving 2 skills, and a student with different values corresponding to those skills. The chance of this student to answer right this item will be the minimum between the 2 values of the skills involved. We name it 'pessimistic approach' since we reduce the students' probability of success to their level in the skill they are weaker.

To generate this kind of data, we use the MinGen() function, with the values we have been working with along this case of use.

We have already generated data according to linear models, so we will start with more complex models. Some of them don't use neither a skills matrix nor a Q-matrix but, as we will see, they can learn from them to generate the information needed.

That said, the first non-linear models used in this case of use are the DINA/DINO models. That stands for Deterministic Input Noisy And/Or. In those models, the Q-Matrix is used but the skills matrix becomes a skills mastery matrix. To get this skills mastery matrix, we will use the original skills matrix and we will consider a skill as mastered for values above 0.5. So this new matrix is a {0,1} matrix where a 1 means that the student i masters the skill j.

Once it has been explained how this new matrix is obtained, we will go for the DINA (And) model. In this model, if an item requires 2 skills, both must be mastered to succeed it. However, there are two noise parameters that must be explained: the slip and guess factors. On the one hand, even if the 2 skills involved are not mastered, the student will have a probability of answering right the item corresponding to the guess factor. On the other hand, being both skills mastered, there is a probability of failure corresponding to the slip parameter.

The DINO model is very similar to the DINA model but, to succeed an item, the mastery of one of the skills required is enough to succeed an item. The slip and guess noise is also present in this model.

We will generate this data using the dinGen() function. We introduce the parameters used along the document and the skills matrix so that the mastery matrix is generated according to it. We first introduce 'DINA' as model. By default, the slip and guess parameters are setted to 0.2.

resultDinA <- dinGen(its = 18,sts = 10, rank = 4 ,q=qMatrix, skills = skills, model = 'DINA')

Once we have understood how the DINA/DINO models work, the next model used is IRT. This model doesn't use neither a Q-Matrix nor a skills matrix. But, as we will see, the function used can learn from a results matrix to generate the parameters needed.

As we know, according to the IRT model, the probability of answering right an item $X_{j}$ will depend on an ability factor $\theta$, the item difficulty $b_{j}$ and a discrimination parameter $a_{j}$:

$$P(X_{j}=1 \ |\ \theta) = \frac{1}{1 + e^{-a_{j}(\theta - b_{j})}}$$

So to generate data according to this model we will use the irtGen() function. As it has been mentioned, this function can learn from a results matrix to predict the students' ability factors and the items' difficulties. So we use the results matrix coming from the pessimistic approach, for example. A slip and guess noise is introduced by default with values of 0.2 for both.

The last model used is the POKS model (Partial Order Knowledge Structure). In this model a POKS matrix is needed. This matrix will determine the item's knowledge order. So, if there are two related items but one is harder than the other, if a student succeed the most difficult, he will answer right the easiest with a strong probability. Supposing that A and B are connected and A is harder than B, if the student has answered right A, his probability of succeeding B will be: $P(B | A) > p_{1}$.

The first step will be to generate this matrix about the dependencies between items. While building this matrix, there are dependencies that can be represented as trees. An example of a tree would be as follows:

$$\{1\rightarrow2,1\rightarrow3,1\rightarrow4\}$$

As we can see, succeeding the item 1, increase the probability of succeeding 2,3 and 4. There are 3 dependencies and the depth of the tree is 1. However, there can be other kind of trees as the one bellow:

$$\{1\rightarrow2,2\rightarrow3,3\rightarrow4\}$$

In this case, the depth is 3 and indirect dependencies appears since there is a link between the items 1 and 2 and between the items 2 and 3, so there is an indirect dependency between items 1 and 3. So the tree including this dependencies would be as follows:

$$\{1\rightarrow2,1\rightarrow3,1\rightarrow4,2\rightarrow3,2\rightarrow4,3\rightarrow4\}$$

In order to generate this matrix, we will use the poksMatrixGen() function. In this function, we can define the number of independent trees, the total number of dependencies in the matrix and if we want a matrix with indirect dependencies or not. We build a matrix with 15 dependencies, 3 independent trees and indirect dependencies.

poks.matrix <- poksMatrixGen(its = 18, depNum = 15 ,trees = 3, indirect.dependencies = TRUE)

We plot now the generated matrix.

oimage(t(poks.matrix),xlab='Succeeding this item', ylab='Increases probability of succeeding this item')

In our case, the trees' structure is the next:

$$\{1\rightarrow2,1\rightarrow4,1\rightarrow6,2\rightarrow3,4\rightarrow5\}$$

$$\{7\rightarrow8,7\rightarrow11,8\rightarrow9,9\rightarrow10,11\rightarrow12\}$$

$$\{13\rightarrow14,13\rightarrow17,14\rightarrow15,15\rightarrow16,17\rightarrow18\}$$

At this point, we have generated data according to all the models offered by the library. However, this case of use goes a little bit farther. The point now is to analize the predictive performance of each model given an underlying model. That means that, given a results matrix generated according to a specific model, we will see how the rest of models are capable to predict the output.

We are not doing it for all the models but we'll give a couple of clearifying examples. We'll use as underlying models the DINA and DINO. So we first generate the data we need. We will work with a set of 150 students, 200 items, no noise and with a probability ($p_{1}$) of 1 for the POKS. This is not realistic but it will help us to better understand.

We generate a Q-Matrix of 200 items with 4 skills involved:

However, while generating these graphs, it's not also interesting to see which one predicts the best, but also to see how the rest of models respond. As studied by [Beheshti, B. and Desmarais, M.C. (2014). Assessing Model Fit With Synthetic vs. Real Data. Polytechnique Montreal.] , these kind of graphs could represent a 'signature'. While working with real data we don't know what the underlying model is. So we could analise how the different models predict this data. After this, we could compare the 'signatures' generated using synthethic data, where we know the underlying model, and maybe identify the underlying model of the real data.
